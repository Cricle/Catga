核心接口

ICatgaMediator

核心

Catga 的核心中介者接口，用于发送命令、查询和发布事件。支持 CQRS 模式，提供类型安全的消息传递。

Task<CatgaResult<TResponse>> SendAsync<TResponse>(IRequest<TResponse> request, CancellationToken cancellationToken = default)
发送命令或查询请求，返回类型化的结果
参数:
request 实现 IRequest<TResponse> 的请求对象
cancellationToken 取消令牌
Task<CatgaResult> SendAsync(IRequest request, CancellationToken cancellationToken = default)
发送无返回值的命令
Task PublishAsync<TEvent>(TEvent @event, CancellationToken cancellationToken = default) where TEvent : IEvent
发布事件到所有订阅者
// 发送命令
var command = new CreateOrderCommand(orderId, customerId);
var result = await mediator.SendAsync(command);

if (result.IsSuccess)
{
    Console.WriteLine($"订单创建成功: {result.Value}");
}

// 发布事件
var @event = new OrderCreatedEvent(orderId);
await mediator.PublishAsync(@event);

IPipelineBehavior<TRequest, TResponse>

核心

管道行为接口，用于在请求处理前后添加横切关注点（如日志、验证、缓存）。

Task<CatgaResult<TResponse>> HandleAsync(TRequest request, RequestHandlerDelegate<TResponse> next, CancellationToken cancellationToken)
处理请求并调用下一个管道行为
public class LoggingBehavior<TRequest, TResponse> 
    : IPipelineBehavior<TRequest, TResponse>
    where TRequest : IRequest<TResponse>
{
    private readonly ILogger<LoggingBehavior<TRequest, TResponse>> _logger;

    public async Task<CatgaResult<TResponse>> HandleAsync(
        TRequest request, 
        RequestHandlerDelegate<TResponse> next, 
        CancellationToken cancellationToken)
    {
        _logger.LogInformation("处理请求: {RequestName}", typeof(TRequest).Name);
        var result = await next();
        _logger.LogInformation("请求完成: {Success}", result.IsSuccess);
        return result;
    }
}

消息传输

IMessageTransport

传输

统一的消息传输接口，支持发布/订阅、点对点、RPC 等多种消息模式。实现包括 InMemory、NATS、Redis。

Task PublishAsync<T>(string topic, T message, MessageOptions? options = null, CancellationToken cancellationToken = default)
发布消息到主题（发布/订阅模式）
Task SendAsync<T>(string queue, T message, MessageOptions? options = null, CancellationToken cancellationToken = default)
发送消息到队列（点对点模式）
Task<IDisposable> SubscribeAsync<T>(string topic, Func<T, MessageContext, Task> handler, CancellationToken cancellationToken = default)
订阅主题消息
// 发布消息
await transport.PublishAsync("orders.created", new OrderCreatedEvent
{
    OrderId = orderId,
    Timestamp = DateTime.UtcNow
});

// 订阅消息
var subscription = await transport.SubscribeAsync<OrderCreatedEvent>(
    "orders.created",
    async (message, context) =>
    {
        Console.WriteLine($"收到订单: {message.OrderId}");
        await ProcessOrderAsync(message);
    }
);

持久化

IEventStore

持久化

事件存储接口，用于 Event Sourcing。支持保存和加载聚合根的事件流。

Task SaveEventsAsync(string aggregateId, IEnumerable<IEvent> events, long expectedVersion, CancellationToken cancellationToken = default)
保存聚合根的事件，支持乐观并发控制
Task<IEnumerable<IEvent>> LoadEventsAsync(string aggregateId, long fromVersion = 0, CancellationToken cancellationToken = default)
加载聚合根的所有事件
// 保存事件
var events = new[] 
{ 
    new OrderCreatedEvent(orderId),
    new OrderItemAddedEvent(orderId, itemId)
};
await eventStore.SaveEventsAsync(orderId, events, expectedVersion: 0);

// 加载事件
var history = await eventStore.LoadEventsAsync(orderId);
var order = Order.LoadFromHistory(history);

IOutboxStore

持久化

Outbox 模式存储接口，确保消息可靠发送。在数据库事务中保存消息，异步发布。

Task AddAsync(OutboxMessage message, CancellationToken cancellationToken = default)
添加消息到 Outbox
Task<IEnumerable<OutboxMessage>> GetPendingAsync(int batchSize, CancellationToken cancellationToken = default)
获取待发送的消息
Task MarkAsPublishedAsync(string messageId, CancellationToken cancellationToken = default)
标记消息已发布
// 在数据库事务中
using var transaction = await dbContext.Database.BeginTransactionAsync();

// 1. 保存业务数据
order.Status = OrderStatus.Confirmed;
await dbContext.SaveChangesAsync();

// 2. 保存到 Outbox
var message = new OutboxMessage
{
    MessageId = Guid.NewGuid().ToString(),
    MessageType = typeof(OrderConfirmedEvent).AssemblyQualifiedName,
    Payload = JsonSerializer.Serialize(new OrderConfirmedEvent(orderId))
};
await outboxStore.AddAsync(message);

await transaction.CommitAsync();

// 后台服务会自动发布消息

IInboxStore

持久化

Inbox 模式存储接口，确保消息幂等处理。防止重复消费相同的消息。

Task<bool> ExistsAsync(string messageId, CancellationToken cancellationToken = default)
检查消息是否已处理
Task AddAsync(string messageId, CancellationToken cancellationToken = default)
记录消息已处理
// 幂等消息处理
var messageId = context.MessageId;

// 检查是否已处理
if (await inboxStore.ExistsAsync(messageId))
{
    Console.WriteLine("消息已处理，跳过");
    return;
}

// 处理消息
await ProcessMessageAsync(message);

// 记录已处理
await inboxStore.AddAsync(messageId);

序列化

IMessageSerializer

序列化

消息序列化接口，支持多种序列化格式（JSON、MemoryPack）。100% AOT 兼容。

byte[] Serialize<T>(T value)
序列化对象到字节数组
T? Deserialize<T>(byte[] data)
从字节数组反序列化对象
// 使用 JSON 序列化
services.AddCatga()
    .AddJsonSerializer();

// 使用 MemoryPack（更快）
services.AddCatga()
    .AddMemoryPackSerializer();

// 自定义使用
var message = new OrderCreatedEvent(orderId);
var data = serializer.Serialize(message);
var restored = serializer.Deserialize<OrderCreatedEvent>(data);

IPooledMessageSerializer

序列化

池化序列化接口，使用 ArrayPool 减少内存分配。适合高性能场景。

PooledBuffer SerializePooled<T>(T value)
序列化到池化缓冲区
T? DeserializePooled<T>(ReadOnlyMemory<byte> data)
从内存反序列化
// 零分配序列化
using var buffer = pooledSerializer.SerializePooled(message);

// 使用缓冲区
await stream.WriteAsync(buffer.Memory);

// 自动返回池（通过 using）

IDistributedLock

辅助

分布式锁接口，用于协调多实例的并发访问。

Task<IDisposable?> AcquireLockAsync(string key, TimeSpan expiration, CancellationToken cancellationToken = default)
获取分布式锁
// 使用分布式锁
await using var lockHandle = await distributedLock.AcquireLockAsync(
    $"order:{orderId}", 
    TimeSpan.FromSeconds(30)
);

if (lockHandle != null)
{
    // 持有锁，执行关键操作
    await ProcessOrderAsync(orderId);
}
// 自动释放锁

// 生成 ID
var orderId = idGenerator.GenerateId();
